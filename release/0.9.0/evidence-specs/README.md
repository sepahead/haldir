# Task evidence specifications

Files here are reviewed inputs to
`tools/release/generate-task-evidence.py`. They contain task-specific scope and
review prose plus the exact repository, event, branch, workflow paths, job sets,
required successful steps, and output markers. They deliberately do not contain
a commit tree, author, run conclusion, head SHA, job result, log digest, byte or
line count, or generated output path. The generator loads the specification and
both evidence tools from `--tool-commit`, verifies both commits against the
repository-pinned SSH signer, and derives the remaining facts from an exact
GitHub Actions attempt and its coverage-validated remote log archive.

GitHub may return either aggregate job logs alone or the same aggregates plus
redundant per-step copies. The generator validates complete metadata/step
coverage in either source layout, then retains a canonical logical pair
(aggregate log and system log) for every exact job. Numeric ZIP ordering and
redundant per-step layout therefore cannot change generated bytes.

Example after both exact-head workflows finish:

```sh
python3 tools/release/generate-task-evidence.py \
  --spec release/0.9.0/evidence-specs/t002.json \
  --implementation-commit COMMIT_SHA \
  --tool-commit TOOL_COMMIT_SHA \
  --ci-run-id CI_RUN_ID \
  --ci-attempt 1 \
  --formal-run-id FORMAL_RUN_ID \
  --formal-attempt 1
```

Generated files use distinct `*-generated-*` names and never overwrite the
historical closure artifacts. They are canonical JSON plus canonical single-
member gzip and are not hand-edited. Verify any present supplements offline with
`python3 tools/release/verify-task-evidence.py --all-present`.

The T000 and T001 specifications were added with the T002 evidence tool. Their
original closure JSON/logs predate this generator, remain unchanged, and are not
claimed as its output. The T002 closure adds separately named T000/T001
supplements whose `implementation.commit` is the historical signed closure
target and whose `evidence_tool.commit` identifies the signed commit that first
carried the tool and specification.
